   - Direct, firm, results-focused tone

   - Voice-call rules: no markdown, no bullet points, no numbered lists

   - `end_call` function tool (FR9 — graceful termination): decorated with `@function_tool()`, docstring "End the call when the conversation is complete.", waits for `ctx.session.current_speech.wait_for_playout()` if speech exists, then calls `delete_room()` via `get_job_context().api.room.delete_room(api.DeleteRoomRequest(room=...))`

6. **entrypoint function** (correct session ordering for Phase 2 compatibility):
   - `await ctx.connect()`
   - Create `AgentSession` with `llm=openai.realtime.RealtimeModel(voice="shimmer")` (NFR1 — single-hop speech-to-speech)
   - Start session via `asyncio.create_task(session.start(...))` — NOT direct await (preserves Phase 2 SIP insertion point)
   - `RoomInputOptions` with `noise_cancellation=noise_cancellation.BVCTelephony()` (NFR2 — telephony-optimized)
   - Comment marking where Phase 2 SIP participant creation will go
   - `await session_started`

7. **Main block**: `cli.run_app(WorkerOptions(entrypoint_fnc=entrypoint, agent_name="accountability-buddy"))`

Anti-patterns to avoid (from research):
- Do NOT use `await session.start()` directly — use `asyncio.create_task()` for Phase 2 compatibility
- Do NOT use `BVC()` — must be `BVCTelephony()` for PSTN audio
- Do NOT call `generate_reply()` inside the end_call tool body — causes race conditions
- Do NOT use `session.aclose()` for hangup — only `delete_room()` reliably disconnects
