<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure Twilio SIP trunk termination and credential-list auth</name>
  <action>
The user already has a Twilio SIP trunk and phone number. Configure the existing trunk for outbound calling:

1. **Set Termination URI** on the existing Twilio trunk:
   - Go to Twilio Console -> Elastic SIP Trunking -> Manage -> Trunks -> [your trunk] -> Termination
   - Set the Termination SIP URI (e.g., `your-trunk-name.pstn.twilio.com`)
   - Note this URI -- you will need the domain part (just `your-trunk-name.pstn.twilio.com`, no `sip:` prefix) for the LiveKit trunk

2. **Create a Credential List** for SIP authentication:
   - Go to Twilio Console -> Elastic SIP Trunking -> Manage -> Trunks -> [your trunk] -> Termination -> Authentication -> Credential Lists
   - Create a new credential list with a username and password of your choice
   - IMPORTANT: Note the username and password -- they are needed for LiveKit outbound trunk creation
   - Associate the credential list with the trunk (select it under Authentication -> Credential Lists)

3. **Verify phone number is associated with trunk:**
   - Go to Twilio Console -> Elastic SIP Trunking -> Manage -> Trunks -> [your trunk] -> Phone Numbers
   - Confirm your Twilio number is listed. If not, add it.

Provide these values when done:
- Twilio termination URI domain (e.g., `your-trunk.pstn.twilio.com`)
- Credential list username
- Credential list password
- Your Twilio phone number (e.g., `+15105550100`)
- Your personal phone number to call (e.g., `+1XXXXXXXXXX`)
  </action>
  <resume-signal>Provide: termination URI domain, credential username, credential password, Twilio phone number, personal phone number</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Create LiveKit outbound SIP trunk and update .env.local</name>
  <files>.env.local</files>
  <action>
Using the values from Task 1, create the LiveKit outbound SIP trunk and update environment variables:

1. **Create outbound trunk JSON file** (`outbound-trunk.json` in project root, temporary):
```json
{
  "trunk": {
    "name": "accountability-buddy-outbound",
    "address": "<termination-uri-domain-from-task-1>",
    "numbers": ["<twilio-phone-number-from-task-1>"],
    "authUsername": "<credential-username-from-task-1>",
    "authPassword": "<credential-password-from-task-1>"
  }
}
```

2. **Run `lk sip outbound create outbound-trunk.json`** to register the trunk with LiveKit Cloud.
   - Copy the `SIPTrunkID` from the output (format: `ST_xxxxxxxxxxxx`)

3. **Verify with `lk sip outbound list`** -- confirm the trunk appears

4. **Add to `.env.local`** (append, do not overwrite existing vars):
```
SIP_OUTBOUND_TRUNK_ID=ST_xxxxxxxxxxxx
DEFAULT_PHONE_NUMBER=+1XXXXXXXXXX
```
Where DEFAULT_PHONE_NUMBER is the user's personal phone number (the one the agent will call).

5. **Delete `outbound-trunk.json`** -- it contains credentials and should not persist.

6. **Add `outbound-trunk.json` to `.gitignore`** as a safety measure (if not already ignored by a broader pattern).

IMPORTANT: The `address` field must be just the domain (e.g., `my-trunk.pstn.twilio.com`), NOT a full SIP URI. No `sip:` prefix.
  </action>

  </verify>
