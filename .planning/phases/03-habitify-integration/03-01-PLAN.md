**Create `scripts/habitify_oauth_setup.py`** — a standalone CLI script the user runs once to authorize with Habitify:

1. **Dynamic client registration:**
   - POST to `https://account.habitify.me/reg` with body:
     ```json
     {
       "client_name": "accountability-buddy",
       "redirect_uris": ["http://localhost:8976/callback"],
       "grant_types": ["authorization_code", "refresh_token"],
       "response_types": ["code"],
       "token_endpoint_auth_method": "none"
     }
     ```
   - Extract `client_id` from response.

2. **Authorization code + PKCE flow:**
   - Generate a random `code_verifier` (43-128 chars, URL-safe base64).
   - Compute `code_challenge` = base64url(sha256(code_verifier)), method `S256`.
   - Build authorization URL:
     ```
     https://account.habitify.me/auth?
       response_type=code&
       client_id=<client_id>&
       redirect_uri=http://localhost:8976/callback&
       scope=openid offline_access all&
       code_challenge=<code_challenge>&
       code_challenge_method=S256&
       state=<random_state>
     ```
   - Print the URL and open it in the default browser (`webbrowser.open`).
   - Start a minimal HTTP server on `localhost:8976` to capture the callback.
   - Extract `code` and `state` from the callback query params. Verify state matches.

3. **Token exchange:**
   - POST to `https://account.habitify.me/token` with form-encoded body:
     ```
     grant_type=authorization_code
     code=<authorization_code>
     redirect_uri=http://localhost:8976/callback
     client_id=<client_id>
     code_verifier=<code_verifier>
     ```
   - Extract `access_token` and `refresh_token` from the JSON response.

4. **Store credentials:**
   - Read existing `.env.local` file (if it exists).
   - Append or update `HABITIFY_CLIENT_ID=<client_id>` and `HABITIFY_REFRESH_TOKEN=<refresh_token>`.
   - Print confirmation with next steps.

Use `http.server` from stdlib for the callback server (no extra deps). Use `httpx` (sync client) for HTTP requests. Use `secrets`, `hashlib`, `base64` for PKCE. Use `webbrowser` to open the auth URL.

**Create `habitify_auth.py`** — runtime module for headless token refresh:

```python
from __future__ import annotations

import logging
import os

import httpx

logger = logging.getLogger("accountability-buddy")

TOKEN_URL = "https://account.habitify.me/token"


async def refresh_habitify_token() -> str:
    """Exchange refresh token for a fresh access token.

    Reads HABITIFY_CLIENT_ID and HABITIFY_REFRESH_TOKEN from environment.
    Returns the access token string.
    Raises RuntimeError if credentials are missing or token exchange fails.
    """
    client_id = os.getenv("HABITIFY_CLIENT_ID")
    refresh_token = os.getenv("HABITIFY_REFRESH_TOKEN")

    if not client_id or not refresh_token:
        raise RuntimeError(
            "HABITIFY_CLIENT_ID and HABITIFY_REFRESH_TOKEN must be set in .env.local. "
            "Run: uv run scripts/habitify_oauth_setup.py"
        )

    async with httpx.AsyncClient() as client:
        resp = await client.post(
            TOKEN_URL,
            data={
                "grant_type": "refresh_token",
                "refresh_token": refresh_token,
                "client_id": client_id,
                "scope": "openid offline_access all",
            },
        )
        if resp.status_code != 200:
            raise RuntimeError(
                f"Habitify token refresh failed ({resp.status_code}): {resp.text}"
            )
        data = resp.json()
        logger.info("Habitify access token refreshed successfully")
        return data["access_token"]
```

Keep it minimal — one async function, clear error messages, reads from env vars.
