**1. Create `habitify.py`** — thin async Habitify REST client using `aiohttp` (already a transitive dependency, no install needed):

- `_format_target_date(tz_offset_hours: int) -> str`: Format current time as ISO-8601 with timezone offset for Habitify API. Use `datetime.now(timezone(timedelta(hours=tz_offset_hours)))`. Format as `%Y-%m-%dT%H:%M:%S±HH:00`.
- `async def fetch_journal(api_key: str, target_date: str) -> list[dict]`: Call `GET {HABITIFY_BASE}/journal` with `Authorization: {api_key}` header and `target_date` query param. Return `data.get("data", [])`. Raise on HTTP error.

Use `HABITIFY_BASE = "https://api.habitify.me"`. Keep this file minimal — three functions max. Use `logging.getLogger("accountability-buddy")` for consistency with agent.py.

**2. Modify `agent.py` entrypoint** — add pre-call reasoning LLM stage between connect and SIP dial:

a) Add new env vars at module level:
```python
habitify_api_key = os.getenv("HABITIFY_API_KEY")
tz_offset = int(os.getenv("TZ_OFFSET_HOURS", "1"))
```

b) In `entrypoint()`, AFTER `await ctx.connect()` and phone number parsing, BEFORE creating `AgentSession`:

- Compute `target_date` using `_format_target_date(tz_offset)`
- Fetch today's habits: `habits = await fetch_journal(habitify_api_key, target_date)`
- Also fetch yesterday's and day-before-yesterday's habits (compute dates with `timedelta`) for pattern detection. Use separate calls — the Journal API only takes one date at a time.
- If `habitify_api_key` is not set or fetch fails, log an error and call `ctx.shutdown()` — do NOT start a call with no habit data.
- Log the count: `logger.info(f"Fetched {len(habits)} habits from Habitify")`

c) Pre-call reasoning LLM call:
- Use `openai.AsyncOpenAI()` (the `openai` Python package is already installed as a dependency of `livekit-plugins-openai`). Instantiate it once.
- Call `client.chat.completions.create()` with model `"gpt-4o-mini"` (cheap, fast, sufficient for reasoning about habit data).
- System prompt for the reasoning LLM:

```
You are preparing a briefing for an accountability coach who is about to call the user.
Analyze the habit data and produce a concise briefing.

For each habit due today:
- Name it and note if it's already completed or still pending
- If it's a goal-based habit, note the target (e.g., "5km run")
- If the user missed this habit yesterday or the day before, flag the streak break

Identify patterns:
- Multiple days of skipping the same habit = escalate
- Already completed habits = brief acknowledgment only
- Goal-based habits = coach should ask for the specific number

Output a natural-language briefing the coach can use directly. No JSON, no bullet formatting — this will be injected into a voice agent's system prompt.
```

- User message: JSON-serialize the multi-day habit data as `{"today": [...], "yesterday": [...], "day_before": [...]}`.
- Extract `response.choices[0].message.content` as the `briefing` string.
