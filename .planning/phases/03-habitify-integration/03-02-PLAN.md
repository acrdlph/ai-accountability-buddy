  <action>
Create `habitify_briefing.py` — an agentic tool-calling loop that connects to the Habitify MCP server and autonomously investigates the user's habit situation before the voice call.

**Architecture:**

The function `generate_briefing(access_token: str) -> str` does the following:

1. **Connect to Habitify MCP server via SSE:**
   - Use the `mcp` Python SDK (`from mcp.client.sse import sse_client` and `from mcp import ClientSession`) to connect to `https://mcp.habitify.me/mcp`.
   - Pass the Bearer token as a header.
   - Call `session.initialize()` then `session.list_tools()` to discover available tools.
   - Convert MCP tool schemas to OpenAI function tool format for the chat completions API.

   Note: The `mcp` Python package should already be available as a transitive dependency of `livekit-agents`. Verify with `uv run python -c "from mcp.client.sse import sse_client"`. If not available, add `"mcp[cli]>=1.0"` to pyproject.toml dependencies and `uv sync`.

2. **Run agentic tool-calling loop with OpenAI chat completions:**
   - System prompt instructs the LLM to analyze the user's habit situation:
     ```
     You are a habit analyst preparing a briefing for an accountability coach.
     Your job: investigate the user's habit data and produce a concise briefing.

     Steps:
     1. Fetch today's habits to see what's due and what's done
     2. Fetch the last 3-5 days to identify patterns
     3. Look for: broken streaks, habits being slacked on, consistent wins
     4. Produce a briefing with:
        - List of today's habits with status (due, completed, in_progress)
        - Pattern observations (e.g. "skipped meditation 3 days in a row")
        - Suggested talking points for the accountability coach
        - Which habits to celebrate, which to push on

     Use the list-habits-by-date tool with different dates to gather data.
     Today's date is provided in the first user message.
     When you have enough data, produce your final briefing.
     ```
   - Use `openai.AsyncOpenAI()` (the openai Python package, not LiveKit's plugin) for the chat completions API.
   - Model: `gpt-4o-mini` (cheap, fast, good enough for data analysis — NOT the realtime model).
   - Loop:
     ```python
     messages = [system_prompt, user_message_with_today_date]
     while True:
         response = await client.chat.completions.create(
             model="gpt-4o-mini",
             messages=messages,
             tools=openai_tools,  # MCP tools converted to OpenAI format
         )
         choice = response.choices[0]
         if choice.finish_reason == "stop":
             break  # LLM produced final briefing
         if choice.finish_reason == "tool_calls":
             # Execute each tool call against the MCP server
             for tool_call in choice.message.tool_calls:
                 result = await mcp_session.call_tool(
                     tool_call.function.name,
                     json.loads(tool_call.function.arguments),
                 )
                 messages.append(tool_result_message)
             messages.append(choice.message)  # assistant message with tool_calls
     ```
   - Max iterations: 15 (safety limit to prevent infinite loops).
   - Return the final text content from the LLM as the briefing string.

3. **MCP tool schema conversion:**
   - MCP tools have `inputSchema` (JSON Schema). OpenAI expects `{"type": "function", "function": {"name": ..., "description": ..., "parameters": ...}}`.
   - Write a helper `_mcp_tools_to_openai(mcp_tools)` that converts the format.
   - Only include tools relevant to reading: `list-habits-by-date`. Exclude write tools from the briefing agent (writes happen during the voice call, not pre-call).

4. **Error handling:**
   - If MCP connection fails, log error and return a fallback briefing: "Could not connect to Habitify. Proceed with a general check-in."
   - If the LLM loop exceeds max iterations, return whatever analysis has been gathered so far.
   - Wrap the entire function in try/except to never crash the entrypoint.

**Key details:**
- Use `from datetime import date` and pass `date.today().isoformat()` as the user message so the LLM knows today's date.
- The SSE client connection should use `headers={"Authorization": f"Bearer {access_token}"}`.
- Log the number of tool calls made and the final briefing length for debugging.
- The MCP `call_tool` result comes back as a `CallToolResult` with `content` (list of content blocks). Extract text from `content[0].text`.
