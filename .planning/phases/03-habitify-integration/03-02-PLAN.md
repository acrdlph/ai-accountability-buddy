<task type="auto">
  <name>Task 2: Implement log_habit function tool and _find_habit on AccountabilityAgent</name>
  <files>agent.py</files>
  <action>
Add two methods to `AccountabilityAgent`:

**1. `_find_habit(self, habit_name: str) -> dict | None`** (private helper, NOT a tool):
- Case-insensitive substring match against `self.habits`
- For each habit in `self.habits`, check if `habit_name.lower()` is in `habit["name"].lower()`
- If exactly one match, return it
- If multiple matches, return the shortest-name match (most specific)
- If no match, return `None`
- This handles the LLM abbreviating names (e.g., "meditation" matches "Morning Meditation (10 min)")

**2. `@function_tool()` `async def log_habit(self, context: RunContext, habit_name: str, value: float | None = None) -> str`**:

Docstring (this is what the LLM sees — make it clear):
```python
"""Log a habit as completed in Habitify. For goal-based habits, include the numeric value achieved.

Args:
    habit_name: The name of the habit to log (can be partial match).
    value: For goal-based habits, the numeric value achieved (e.g., 5.0 for 5km). Not needed for simple yes/no habits.
"""
```

Implementation:
- Call `self._find_habit(habit_name)` — if `None`, return `f"Could not find habit '{habit_name}'. Available habits: {', '.join(h['name'] for h in self.habits)}"`
- Branch on `habit.get("goal")`:
  - **If goal exists (goal-based habit):**
    - If `value is None`, return `"This is a goal-based habit. Please ask the user for the specific number before logging."`
    - Call `await add_log(api_key=self.api_key, habit_id=habit["id"], value=value, unit_type=habit["goal"]["unit_type"], target_date=self.target_date)`
  - **If no goal (simple habit):**
    - Call `await update_status(api_key=self.api_key, habit_id=habit["id"], status="completed", target_date=self.target_date)`
- Return success/failure message: `f"Logged '{habit['name']}' as complete"` or `f"Failed to log '{habit['name']}' — API error"`
